# Session 2026-04-25: CLI, Version Bumping, Component Docs

## 1. CLI Registry Path Resolution (FIXED)

**Problem:** `lunar add button` failed with "Local registry not found" when CLI installed globally.

**Root Cause:** CLI used CommonJS `__dirname`, doesn't work in ESM modules.

**Solution:**
- Updated `packages/core/src/cli-utils.ts` to use `import.meta.url` with `fileURLToPath()`
- Added post-build hook in `packages/core/tsup.config.ts` to copy registry to `dist/registry`
- Updated `packages/cli/src/commands/list.ts` to fallback to bundled registry path

**Files Changed:**
- `packages/core/src/cli-utils.ts` - ESM path resolution
- `packages/core/tsup.config.ts` - Registry bundling with `onSuccess` hook
- `packages/cli/src/commands/list.ts` - Fallback registry discovery
- `packages/core/package.json` - Removed `src/registry` from files (now in dist)

**Result:** Registry bundled with core package, discoverable when installed globally ✓

## 3. Version Bumping Script Fix

**Problem:** User's semantic versioning v[version].[major].[minor] was misaligned with script v[major].[minor].[patch].

When user said "release: major" (bump middle digit v0.0.20 → v0.1.0), script gave wrong result.

**Understanding:**
- User format: v[version].[major].[minor]
  - First digit: product version line
  - Second digit: major feature
  - Third digit: minor/patch
- Script was treating first digit as semver "major"

**Solution:**
1. Renamed script variables to match actual use:
   - parts.major → parts.version
   - parts.minor → parts.major
   - parts.patch → parts.minor

2. Added 4th bump type "version" for bumping first digit:
   - `release: version` → v0.1.0 → **v1.0.0**
   - `release: major` → v0.0.20 → **v0.1.0**
   - `release: minor` → v0.1.0 → **v0.1.1**

3. Updated `release.yml` to map commit message correctly

**Files Changed:**
- `scripts/bump-versions.ts` - Variable renaming + version type handling
- `.github/workflows/release.yml` - Command mapping and security hardening

**Result:** Version bumping now matches user's semantic expectations ✓
