Lunar Kit provides a powerful CLI tool for scaffolding, managing components, and generating code. This guide covers all available commands.

## Installation

Install the CLI globally or use `npx`:

<DynamicCodeBlock lang="bash" code={`npm install -g @lunar-kit/cli
# or
npx @lunar-kit/cli <command>`} />

## Core Commands

### `lunar init`

Initialize Lunar Kit in your project.

<DynamicCodeBlock lang="bash" code={`lunar init`} />

Sets up:
- Configuration file (`kit.config.json`)
- Directory structure
- Initial dependencies

### `lunar doctor`

Check your Lunar Kit setup and configuration.

<DynamicCodeBlock lang="bash" code={`lunar doctor`} />

Verifies:
- Node.js version
- Package manager availability
- Lunar Kit project detection
- Core packages installation
- Required dependencies

### `lunar install | lunar i`

Install packages with progress bar.

<DynamicCodeBlock lang="bash" code={`lunar install dayjs axios
lunar i -D @types/node
lunar i react-native-reanimated`} />

Supports:
- Multiple packages
- Dev dependencies (`-D` flag)
- Any npm package

### `lunar upgrade`

Update Lunar Kit packages to latest version.

<DynamicCodeBlock lang="bash" code={`lunar upgrade`} />

### `lunar migrate`

Run breaking change migrations between versions.

<DynamicCodeBlock lang="bash" code={`lunar migrate`} />

## Add Command

### `lunar add`

Add components or features to your project.

<DynamicCodeBlock lang="bash" code={`lunar add <component>
lunar add <subcommand>`} />

#### Subcommands

| Subcommand | Description |
|------------|-------------|
| `locale` | Add a translation entry to all locale files |
| `locale update` | Bulk-add translations for a specific locale |
| `<component>` | Add a UI component (button, dialog, card, etc.) |

#### Examples

<DynamicCodeBlock lang="bash" code={`# Add UI components
lunar add button
lunar add dialog
lunar add card
lunar add input

# Add localization
lunar add locale
lunar add locale update`} />

## Generate Command

### `lunar generate | lunar g`

Generate code modules, services, components, and more.

<DynamicCodeBlock lang="bash" code={`lunar g <schematic> [path]`} />

#### Module Generation

| Schematic | Alias | Description |
|-----------|-------|-------------|
| `mod` | `mod` | Generate a module |
| `tabs` | `tabs` | Generate a tabs module |

<DynamicCodeBlock lang="bash" code={`lunar g mod auth/login
lunar g tabs settings`} />

#### Module-Scoped Generation

| Schematic | Alias | Description |
|-----------|-------|-------------|
| `mod:view` | `mod:vi` | Generate a view in module |
| `mod:component` | `mod:co` | Generate a component in module |
| `mod:store` | `mod:st` | Generate a store in module |
| `mod:hook` | `mod:ho` | Generate a hook in module |
| `mod:api` | `mod:ap` | Generate API layer in module |
| `mod:type` | `mod:tp` | Generate TypeScript types in module |
| `mod:test` | `mod:ts` | Generate test file in module |

<DynamicCodeBlock lang="bash" code={`lunar g mod:vi auth/login-screen
lunar g mod:api auth/auth-api
lunar g mod:st auth/auth-store
lunar g mod:ho auth/use-auth
lunar g mod:type auth/user
lunar g mod:test auth/auth.test`} />

#### Global Generation

| Schematic | Alias | Description |
|-----------|-------|-------------|
| `component` | `co` | Generate a global component |
| `hook` | `ho` | Generate a global hook |
| `store` | `st` | Generate a global store |
| `service` | `sv` | Generate global service |
| `type` | `ty` | Generate global TypeScript types |
| `config` | `cf` | Generate config file |
| `test` | `te` | Generate global test helper/setup |

<DynamicCodeBlock lang="bash" code={`lunar g component Button
lunar g hook useAuth
lunar g store authStore
lunar g service api
lunar g type User
lunar g config env
lunar g test setup`} />

#### Navigation Generation

| Schematic | Alias | Description |
|-----------|-------|-------------|
| `nav:stack` | `n:stack` | Generate stack navigator |
| `nav:tab` | `n:tab` | Generate tab navigator |
| `nav:drawer` | `n:drawer` | Generate drawer navigator |

<DynamicCodeBlock lang="bash" code={`lunar g nav:stack RootNavigator
lunar g nav:tab BottomTabs
lunar g nav:drawer MainDrawer`} />

#### Localization Generation

| Schematic | Alias | Description |
|-----------|-------|-------------|
| `locale` | `lo` | Generate a new locale file |

<DynamicCodeBlock lang="bash" code={`lunar g locale zh
lunar g locale fr
lunar g locale es`} />

## Tips & Tricks

- Use aliases for faster typing: `lunar g mod:vi` instead of `lunar g mod:view`
- Chain commands: `lunar add button && lunar add dialog`
- Check help anytime: `lunar add help`, `lunar g help`
- Install dev dependencies with `-D`: `lunar i -D @types/node`
