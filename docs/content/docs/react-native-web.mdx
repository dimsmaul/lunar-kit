# React Native Web in Docs

This documentation site uses **React Native Web** to display live React Native component previews directly in Fumadocs.

## How It Works

1. **React Native Web** converts React Native components to web standards (DOM, CSS)
2. **Tailwind CSS** is used for styling the docs site
3. Components are rendered in isolated preview containers

## Architecture

React Native Web is set up with the following configuration:

**next.config.mjs**
- Maps `react-native` to `react-native-web` for browser compatibility
- Configures webpack to recognize `.web.js/.web.tsx` extensions

**Component Preview**
- `ComponentPreview` wrapper component for isolated previews
- Located in `src/components/component-preview.tsx`

**Styling**
- Base React Native Web styles are available globally
- Tailwind CSS handles document styling
- Inline styles work with React Native Web

## Writing Documentation

### 1. Create a new MDX file

Create `content/docs/components/my-component.mdx`:

```mdx
---
title: My Component
description: Component description
---

# My Component

Component overview here.

## Usage

```tsx
import { MyComponent } from '@lunar-kit/core';
import { View } from 'react-native';

export function Example() {
  return (
    <View>
      <MyComponent />
    </View>
  );
}
```

## Examples

Show different use cases and variants...
```

### 2. Component Preview

Use the `ComponentPreview` component for isolated previews:

```tsx
import { ComponentPreview } from '@/components/component-preview';

<ComponentPreview 
  title="Example"
  description="Description"
  height={200}
>
 {/* Preview content */}
</ComponentPreview>
```

### Props

- `title` (optional): Preview heading
- `description` (optional): Preview description
- `children`: React content to render
- `height` (optional): Container height in pixels (default: 400)
- `background` (optional): Background color (default: #f5f5f5)
- `className` (optional): CSS wrapper classes

## React Native Web Capabilities

### ✅ What Works
- Core components: `View`, `Text`, `Pressable`, `ScrollView`
- Inline styles with React Native style props
- Flexbox layouts and styling
- Touch handlers (`onPress`, etc.)
- Custom components built from React Native basics
- CSS classes on supported components

### ❌ What Doesn't Work
- Native modules (camera, notifications, sensors, etc.)
- Platform-specific APIs
- Native animations or complex Reanimated features
- Navigation or routing
- File access or device APIs

## Styling

### React Native Styles

Use inline `style` props for React Native-specific styling:

```tsx
<View style={{
  flex: 1,
  flexDirection: 'row',
  justifyContent: 'center',
  alignItems: 'center',
  gap: 16,
  paddingHorizontal: 20,
}}>
  <Text style={{ fontSize: 18, fontWeight: 'bold' }}>
    Hello
  </Text>
</View>
```

### Flexbox Layout

React Native Web uses flexbox by default:

```tsx
// Column layout (default)
<View style={{ flex: 1, gap: 8 }}>
  <Text>Item 1</Text>
  <Text>Item 2</Text>
</View>

// Row layout
<View style={{ flexDirection: 'row', gap: 8 }}>
  <Text>Left</Text>
  <Text>Right</Text>
</View>
```

## Troubleshooting

**Build fails with "Can't resolve 'react-native'"**
- Don't import React Native components directly in MDX frontmatter
- Only import from `@/components/component-preview`
- React Native imports should be in `<code>` blocks or separate files

**Preview not rendering**
- Check browser DevTools console for errors
- Verify component logic is correct
- Test with simpler components first

**Styles not applying**
- Use `style` prop for React Native-specific styling
- Check for conflicting CSS from Tailwind
- React Native Web may have limited CSS support for some properties

## Performance

- Previews are isolated in containers
- No impact on page load for non-preview content
- CSS is cached by browser
- Keep individual previews simple
