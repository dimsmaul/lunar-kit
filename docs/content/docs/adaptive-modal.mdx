# Using Adaptive Modal

The `@lunar-primitive/adaptive-modal` is a powerful primitive that enables **universal components** that work seamlessly on both web and native platforms.

## What is Adaptive Modal?

Adaptive Modal is a platform-aware container that:

- **Detects platform** using React Native's `Platform.OS`
- **On Web**: Renders using React Portal to DOM
- **On Native**: Renders using native Modal component  
- **Unified API**: Same component works everywhere

## Installation

```bash
# Already available in lunar-kit monorepo
import { AdaptiveModal, Portal } from '@lunar-primitive/adaptive-modal';
```

## Basic Usage

```tsx
import { AdaptiveModal } from '@lunar-primitive/adaptive-modal';
import { View, Text, Pressable } from 'react-native';
import { useState } from 'react';

export function MyModalExample() {
  const [visible, setVisible] = useState(false);

  return (
    <View>
      <Pressable onPress={() => setVisible(true)}>
        <Text>Open Modal</Text>
      </Pressable>

      <AdaptiveModal
        visible={visible}
        onDismiss={() => setVisible(false)}
      >
        <View style={{ padding: 20 }}>
          <Text>This works on iOS, Android, and Web!</Text>
        </View>
      </AdaptiveModal>
    </View>
  );
}
```

## Platform Detection

The component automatically detects the platform:

```tsx
import { Platform } from 'react-native';

export function MyComponent() {
  if (Platform.OS === 'web') {
    // Running in browser via React Native Web
    // AdaptiveModal uses Portal
  } else {
    // Running on iOS or Android
    // AdaptiveModal uses native Modal
  }
}
```

## In Fumadocs (Web Documentation)

When components are rendered in Fumadocs:

1. **RNWebProvider** initializes React Native Web
2. **Sets Platform.OS = 'web'** at runtime
3. **AdaptiveModal detects this** and uses Portal
4. **Portal renders to DOM** via React.createPortal

```tsx
// This happens automatically in @lunar-kit component docs
<RNWebProvider>
  <ComponentPreview>
    <YourComponent /> {/* Platform.OS is 'web' */}
  </ComponentPreview>
</RNWebProvider>
```

## Portal Component

Use Portal directly for custom rendering:

```tsx
import { Portal } from '@lunar-primitive/adaptive-modal';
import { View } from 'react-native';

export function CustomPortal() {
  return (
    <Portal>
      <View style={{ position: 'absolute', top: 0, left: 0 }}>
        {/* Renders outside normal component tree */}
      </View>
    </Portal>
  );
}
```

**On Web**: Uses `React.createPortal` to DOM body
**On Native**: Renders inline (native doesn't support portals)

## Creating Universal Components

Follow this pattern for universal lunar-kit components:

```tsx
// src/components/my-dialog.tsx
import { AdaptiveModal, Portal } from '@lunar-primitive/adaptive-modal';
import { View, Text, Pressable } from 'react-native';
import { useCallback, useState } from 'react';

export interface MyDialogProps {
  title: string;
  description?: string;
  onConfirm: () => void;
  onCancel: () => void;
}

export function MyDialog({
  title,
  description,
  onConfirm,
  onCancel,
}: MyDialogProps) {
  const [visible, setVisible] = useState(false);

  const handleConfirm = useCallback(() => {
    onConfirm();
    setVisible(false);
  }, [onConfirm]);

  return (
    <>
      <Pressable onPress={() => setVisible(true)}>
        <Text>{title}</Text>
      </Pressable>

      <AdaptiveModal visible={visible} onDismiss={onCancel}>
        <View style={{ padding: 20, backgroundColor: 'white' }}>
          <Text style={{ fontSize: 18, fontWeight: 'bold' }}>{title}</Text>
          {description && <Text style={{ marginTop: 8 }}>{description}</Text>}
          
          <View style={{ flexDirection: 'row', marginTop: 20, gap: 10 }}>
            <Pressable onPress={onCancel} style={{ flex: 1 }}>
              <Text>Cancel</Text>
            </Pressable>
            <Pressable onPress={handleConfirm} style={{ flex: 1 }}>
              <Text>Confirm</Text>
            </Pressable>
          </View>
        </View>
      </AdaptiveModal>
    </>
  );
}
```

## Documentation in Fumadocs

To document your adaptive component in Fumadocs:

```mdx
# My Dialog Component

<ComponentPreview
  title="Dialog Example"
  description="Works on web and native"
>
  {/* Component renders with Platform.OS = 'web' via RNWebProvider */}
</ComponentPreview>

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| title | string | Yes | Dialog title |
| description | string | No | Dialog description |
| onConfirm | () => void | Yes | Confirm callback |
| onCancel | () => void | Yes | Cancel callback |
```

## Troubleshooting

### Modal not appearing in web docs

**Problem**: AdaptiveModal renders nothing in Fumadocs

**Solution**: Ensure `RNWebProvider` wraps the component
```tsx
<RNWebProvider>
  <ComponentPreview>
    <MyComponent />
  </ComponentPreview>
</RNWebProvider>
```

### Platform.OS is 'ios' or 'android'

**Problem**: Component renders as native Modal instead of Portal

**Solution**: Only happens on actual iOS/Android devices. In Fumadocs (web), ensure Platform.OS is detected as 'web'

Check in browser console:
```javascript
// Paste in browser dev console
console.log(require('react-native').Platform.OS) // Should be 'web'
```

### Portal content not visible

**Problem**: Portal content renders but isn't visible

**Solution**: Ensure you have proper z-index and positioning
```tsx
<Portal>
  <View style={{ 
    position: 'absolute', 
    top: 0, 
    left: 0, 
    right: 0, 
    bottom: 0,
    zIndex: 1000,  // Add z-index
  }}>
    {/* Content */}
  </View>
</Portal>
```
