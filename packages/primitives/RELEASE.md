# Lunar Kit Primitives - Release Guide

## Overview

Lunar Kit Primitives uses **Changesets** for independent versioning and publishing. Unlike the core packages (`@lunar-kit/core`, `@lunar-kit/cli`, `create-lunar-kit`) which share the same version, each primitive package (`@lunar-primitive/*`) is versioned and published independently.

## How It Works

### 1. Making Changes

When you make changes to a primitive package, create a changeset:

```bash
bun changeset
```

This will:
- Prompt you to select which packages changed
- Ask for the version bump type (major/minor/patch)
- Generate a changeset file in `.changeset/`

### 2. Changeset File Example

```markdown
---
"@lunar-primitive/adaptive-modal": minor
---

Added new prop `closeOnBackdropPress` to AdaptiveModal component.
```

### 3. Release Process

The release is **automated** via GitHub Actions:

1. **Push to main**: When changesets exist, the workflow creates a version PR
2. **Merge the PR**: Merging the version PR triggers the publish
3. **Automatic publish**: Packages are published to npm independently
4. **GitHub Release**: A GitHub release is created for tracking

### 4. Manual Commands

```bash
# Check changeset status
bun run release:check

# Generate version updates
bun run release:version

# Publish to npm
bun run release:publish
```

## Versioning Strategy

- **Patch** (`0.0.x`): Bug fixes
- **Minor** (`0.x.0`): New features (backwards compatible)
- **Major** (`x.0.0`): Breaking changes

## Package Independence

Each primitive package has its own version:

- `@lunar-primitive/adaptive-modal@1.0.0`
- `@lunar-primitive/bottom-sheet@0.2.5`

This allows:
- ✅ Independent releases (no need to release all packages together)
- ✅ Faster iteration on individual components
- ✅ Clearer changelogs per package
- ✅ Smaller, focused releases

## NPM Tokens

Ensure the following secrets are configured in GitHub:
- `NPM_TOKEN`: npm publish token
- `GITHUB_TOKEN`: Auto-generated by GitHub Actions

## Troubleshooting

### Package didn't publish?
- Check if changeset file exists in `.changeset/`
- Verify package.json has correct name (`@lunar-primitive/*`)
- Check GitHub Actions logs for errors

### Wrong version bump?
- Edit the changeset file before merging version PR
- Use `major`, `minor`, or `patch` as needed

### Need to skip a package?
- Don't include it in the changeset
- Or add to `ignore` in `.changeset/config.json`
